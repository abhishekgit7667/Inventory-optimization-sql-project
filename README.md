# Inventory-optimization-sql-project

📌 Project Overview
This project focuses on Inventory Optimization using SQL.
The objective is to analyze stock levels, demand patterns, and reorder decisions to reduce overstocking and stockouts.
The dataset contains 500 rows of inventory records including product details, demand, stock, and supplier information.

🎯 Business Problem
Companies often face:
-Overstocking (extra capital blocked in inventory)
-Understocking (lost sales due to stockout)
-Poor demand planning
-No clear reorder strategy
This project solves these issues using SQL-based analysis.

🗂 Dataset Information
Columns used:
-product_id
-product_name
-category
-supplier
-warehouse
-stock_quantity
-monthly_demand
-lead_time_days
-unit_cost
-reorder_level
-last_restock_date

🛠 Tools Used
-MySQL
-SQL (Joins, Subqueries, Group By, Aggregations)
-GitHub
